# LinkedIn Post Guideline

## Topic
This post will clarify that context engineering is a much broader and more critical discipline than commonly perceived, extending beyond just prompt engineering.

## Angle
Start by debunking the common misconception that context engineering is merely about writing better prompts. Position context engineering as the strategic curation of an LLM's "short-term memory" within strict token limits, likening it to how a computer uses RAM. Systematically explain the various components that contribute to this context payload, emphasizing the skill required to optimize it.

## Target Audience
Engineers, AI developers, and machine learning practitioners who work with or are learning about large language models and intelligent agents, particularly those who might oversimplify the concept of prompt engineering.

## Key Points to Cover
*   Challenge the idea that prompt engineering is synonymous with context engineering.
*   Define context engineering as the process of carefully selecting and organizing all information an LLM sees to optimize its performance.
*   Explain the various distinct components that comprise an LLM's "context payload" (e.g., system prompt, message history, retrieved information, user preferences, tool schemas).
*   Emphasize the limited nature of an LLM's context window and the need for efficient, relevant information inclusion.
*   Conclude that effective context engineering is about balancing informativeness with conciseness to make the model smart without noise.

## Tone
Authoritative and educational, with a slightly myth-busting opening. It should be clear, direct, and practical, aiming to inform and correct common misunderstandings in an accessible manner.
